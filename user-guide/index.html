User Guide

Install the framework globally; keep each project's context local.

1. Overview

AI Coding OS gives multiple coding Agents the same workflows, memory, and human approval rules. It includes the core and five built-in skills.

2. Global installation

git clone https://github.com/DOS-AI-Tech/AI-Coding-OS.git ~/ai-coding-os
bash ~/ai-coding-os/installer/install.sh
git clone https://github.com/DOS-AI-Tech/AI-Coding-OS.git "$HOME\ai-coding-os"
& "$HOME\ai-coding-os\installer\install.ps1"

Select Claude Code, Codex, Copilot, Cursor, or Qoder. Node/npm is not required.

3. Project initialization

bash ~/ai-coding-os/installer/init.sh /path/to/project

Creates only .ai/memory, domain, project_docs, the dashboard, and thin adapters; global skills are not copied.

4. Daily usage

  1. Open an initialized project and describe one task.
  2. The Agent reads project memory and selects a workflow.
  3. Review and confirm the gate.
  4. After completion, review project_docs, tests, and the task log.

5. Workflows and gates

ScenarioFlow
New productdefine → scope → architecture → plan → build → test → deliver
Changeanalyze → decision gate → plan gate → build → verify
Bugreproduce → root cause → fix plan → regression test

6. Memory, domain, and outputs

.ai/memory/architecture, deployment, current state, work items, logs
domain/project-specific business knowledge
project_docs/requirements, architecture, plans, and test evidence

7. Built-in and Advanced Skills

Five built-in skills cover requirements, architecture, project management, testing, and deployment. Advanced Skill Packs add domain discovery, business modeling, technical analysis, solution design, and expert review workflows.

8. Upgrade

cd ~/ai-coding-os && git pull
bash installer/update.sh

Global updates do not modify project memory.

9. Troubleshooting

Agent cannot see the framework

Run the global installer again, select that Agent, then restart it.

Project has no context

Run project initialization, then complete frozen memory and domain material.
