使用指南

全局安装框架，每个项目只保存自己的上下文。

1. 概述

AI Builder OS 让多个编码 Agent 使用相同的工作流、记忆和人工审批规则。免费框架包含核心能力和五个基础技能。

2. 全局安装

git clone https://github.com/DOS-AI-Tech/AI-Builder-OS.git ~/ai-builder-os
bash ~/ai-builder-os/installer/install.sh
git clone https://github.com/DOS-AI-Tech/AI-Builder-OS.git "$HOME\ai-builder-os"
& "$HOME\ai-builder-os\installer\install.ps1"

选择 Claude Code、Codex、Copilot、Cursor 或 Qoder。无需 Node/npm。

3. 项目初始化

bash ~/ai-builder-os/installer/init.sh /path/to/project

只创建 .ai/memory、domain、project_docs、Dashboard 和薄适配器；不会复制全局技能。

4. 日常使用

  1. 打开已初始化的项目并描述一个任务。
  2. Agent 读取项目记忆并选择工作流。
  3. 检查并确认 Gate。
  4. 完成后检查 project_docs、测试结果和任务日志。

5. 工作流与 Gate

场景流程
新产品定义 → 范围 → 架构 → 计划 → 实现 → 测试 → 交付
变更分析 → 决策 Gate → 计划 Gate → 实现 → 验证
Bug复现 → 根因 → 修复计划 → 回归测试

6. 记忆、领域和输出

.ai/memory/架构、部署、当前状态、工作项和日志
domain/项目专属业务知识
project_docs/需求、架构、计划和测试证据

7. 基础与付费技能

基础技能随免费框架全局安装。付费技能包独立交付，并强制依赖免费框架。

bash ~/ai-builder-os/installer/install-skill.sh <skill-name> /path/to/purchased-pack

8. 升级

cd ~/ai-builder-os && git pull
bash installer/update.sh

全局框架更新不会修改项目记忆或付费技能。

9. 故障排查

Agent 看不到框架

重新运行全局安装并选择该 Agent，然后重启 Agent。

付费技能无法安装

先确认同一个 Agent 已安装免费 ai-builder-os 核心。

项目没有上下文

运行项目初始化，并填写 frozen memory 和 domain 资料。
